# Plantilla SEO y reglas editoriales

Fuente principal: tarea Odoo `#511 Estructura`.

## Uso

Esta plantilla sirve para generar articulos y paginas de spots con consistencia.

## Estructura base

### 1. Title SEO

- formato:
  `keyword principal + beneficio`
- rango recomendado:
  `50-60 caracteres`

### 2. Meta description

- resumen breve
- incluir keyword
- incluir beneficio o razon para clicar

### 3. H1

- un solo H1
- debe contener la keyword principal

### 4. Introduccion

- 100-150 palabras
- explicar que es
- introducir el contexto
- incluir keyword sin forzarla

### 5. H2 de contexto

- que es
- historia o descripcion
- por que es importante

### 6. H2 de experiencia

- que puedes ver
- condiciones
- profundidad
- corrientes
- nivel recomendado

### 7. H2 de conversion

- como visitar este punto en un liveaboard
- desde donde salen los barcos
- que ruta lo incluye

### 8. FAQ

- 4-10 preguntas reales
- resolver objeciones, no rellenar

### 9. CTA final

- paso claro
- pedir fechas, itinerario o consulta

## Longitud orientativa

- spots y articulos:
  `1200-1800 palabras`
- landings comerciales:
  segun necesidad, pero con densidad alta de informacion util

## Interlinking minimo

Cada pieza debe enlazar a:

- pagina pilar principal
- una landing comercial relacionada
- 2 o 3 paginas hermanas del mismo cluster

Ejemplo:

- `Thistlegorm -> Ruta Norte + Abu Nuhas + checklist`
- `Abu Nuhas -> Giannis D + Carnatic + landing del viaje`

## Regla editorial importante

No hacer `blog posts` genericos.

Cada pagina debe ser:

- informativa
- orientada a keyword
- conectada a negocio
- util para mover al usuario al siguiente paso

## Checklist previa a publicar

- keyword principal definida
- intencion definida
- H1 unico
- estructura H2 clara
- CTA presente
- enlazado interno presente
- no canibaliza otra URL existente
