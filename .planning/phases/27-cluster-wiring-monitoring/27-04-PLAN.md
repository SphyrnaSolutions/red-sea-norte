
**New signature**:
```ts
export function buildBreadcrumbItems(
  contentType: string,
  pageTitle: string,
  pageSlug?: string,
  clusterPillar?: { title: string; url: string } | null
): Array<{ name: string; href: string }>
```

**Logic change**: When `clusterPillar` is provided and the current page is NOT the pillar itself (i.e., `pageSlug` differs from the slug in `clusterPillar.url`), replace the middle breadcrumb level:

```ts

// With cluster pillar: Inicio > [Pillar Title] > [Page Title]
const middleItem = clusterPillar
  ? { name: clusterPillar.title, href: clusterPillar.url }
  : { name: displayName, href: `/${contentType}` }
```

Return:
```ts
return [
  { name: 'Inicio', href: '/' },
  middleItem,
  { name: pageTitle, href: pageHref },
]
```

The `Breadcrumbs` component itself needs no changes — it already calls `buildBreadcrumbSchema` on whatever items it receives. The schema update is automatic.

Keep all existing behavior intact when `clusterPillar` is not provided (backward compatible — all existing call sites continue to work without changes).
