<task type="auto">
  <name>Task 1: Rate limiter and POST /api/lead route</name>
  <files>src/lib/rate-limit.ts, src/app/api/lead/route.ts</files>
  <action>
1. Create `src/lib/rate-limit.ts`:
   - In-memory sliding window rate limiter (no external deps needed for this scale)
   - `RateLimitConfig` interface: `{ windowMs: number, maxRequests: number }`
   - `RateLimitResult` interface: `{ allowed: boolean, remaining: number, resetAt: number }`
   - Store: `Map<string, number[]>` mapping IP to array of request timestamps
   - `rateLimit(ip: string, config?: RateLimitConfig): RateLimitResult` function:
     - Default config: `{ windowMs: 60_000, maxRequests: 5 }` (5 per minute)
     - Filter out timestamps older than windowMs
     - If count >= maxRequests, return `{ allowed: false, remaining: 0, resetAt: oldest + windowMs }`
     - Otherwise push current timestamp, return `{ allowed: true, remaining: maxRequests - count, resetAt: ... }`
   - Periodic cleanup: every 5 minutes, delete entries with no recent timestamps to prevent memory leak
   - Export `rateLimit` and `RateLimitResult`

2. Create `src/app/api/lead/route.ts`:
   - Export `POST` handler (Next.js App Router convention: `export async function POST(request: NextRequest)`)
   - Import `NextRequest`, `NextResponse` from `next/server`

   **Step 1 — Rate limiting:**
   - Extract IP from `request.headers.get("x-forwarded-for")` (first IP if comma-separated) or `request.headers.get("x-real-ip")` or fallback `"unknown"`
   - Call `rateLimit(ip)`
   - If not allowed: return `NextResponse.json({ error: "Demasiadas solicitudes. Intente de nuevo en un minuto." }, { status: 429 })` with `Retry-After` header

   **Step 2 — Parse and validate:**
   - Parse request body with `request.json()` (wrap in try/catch for malformed JSON -> 400)
   - Call `validateLeadData(body)` from `@/lib/validation/lead`
   - If validation fails: return `NextResponse.json({ error: "Datos invalidos", fields: result.errors }, { status: 400 })`

   **Step 3 — Build lead data for Odoo:**
   - Call `searchOrCreateUtmSource("Web buceoenelmarrojo.com")` from `@/lib/odoo/client` to get `source_id`
   - Build description field:
     ```
     Certificacion: {certification || "No especificada"}
     Mes preferido: {preferredMonth || "No especificado"}

     --- Consentimiento WhatsApp ---
     Acepto recibir comunicaciones por WhatsApp: Si
     Fecha de consentimiento: {ISO 8601 timestamp}
     ```
   - Build `LeadCreateData` object:
     - `name`: "Lead Web - {data.name}" (prefix for easy CRM filtering)
     - `contact_name`: data.name
     - `email_from`: data.email
     - `phone`: data.phone
     - `description`: built description string above
     - `source_id`: from searchOrCreateUtmSource
     - `medium_id`: false (or omit — let Odoo use default)
     - `website`: "https://buceoenelmarrojo.com"

   **Step 4 — Create lead in Odoo:**
   - Call `createLead(leadData)` from `@/lib/odoo/client`
   - If successful: return `NextResponse.json({ success: true, message: "Solicitud recibida correctamente" }, { status: 201 })`
   - If Odoo call fails: `console.error("[API Lead]", error)`, return `NextResponse.json({ error: "Error al procesar la solicitud. Intente mas tarde." }, { status: 500 })`

   **Security:**
   - Do NOT return internal error details to client
   - Log full error server-side with `[API Lead]` prefix
   - All env vars are server-only (no NEXT_PUBLIC_)

   **Response format (consistent envelope):**
   - Success: `{ success: true, message: string }`
   - Validation error: `{ error: string, fields: Record<string, string> }`
   - Rate limit: `{ error: string }`
   - Server error: `{ error: string }`

Follow project conventions: named export, no semicolons, 2-space indent. Use `export const dynamic = "force-dynamic"` since this is a mutation endpoint (no caching).
