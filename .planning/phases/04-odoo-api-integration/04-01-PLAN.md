<task type="auto">
  <name>Task 1: Odoo XML-RPC client and configuration</name>
  <files>src/lib/odoo/client.ts, src/lib/odoo/types.ts, src/lib/odoo/config.ts, .env.example</files>
  <action>
1. Install `xmlrpc` package: `npm install xmlrpc` and `npm install -D @types/xmlrpc`. This library provides XML-RPC client for Node.js, used for Odoo's external API.

2. Create `src/lib/odoo/types.ts`:
   - `OdooConfig` interface: `{ url: string, db: string, login: string, apiKey: string }`
   - `LeadCreateData` interface with crm.lead fields: `{ name: string, contact_name: string, email_from: string, phone: string, description: string, source_id?: number, medium_id?: number, website?: string }`
   - `OdooAuthResult` type: `number` (uid returned by authenticate)
   - Export all types

3. Create `src/lib/odoo/config.ts`:
   - `getOdooConfig()` function that reads env vars: `ODOO_URL`, `ODOO_DB`, `ODOO_LOGIN`, `ODOO_API_KEY`
   - Throw descriptive error if any var is missing (e.g., "Missing ODOO_URL environment variable")
   - Return `OdooConfig` object
   - These are server-only vars (no NEXT_PUBLIC_ prefix)

4. Create `src/lib/odoo/client.ts`:
   - Import `xmlrpc` library (createClient for HTTP, createSecureClient for HTTPS)
   - `authenticate(config: OdooConfig): Promise<number>` — calls `xmlrpc/2/common` endpoint with `authenticate(db, login, apiKey, {})`, returns uid
   - `executeKw(config: OdooConfig, uid: number, model: string, method: string, args: unknown[]): Promise<unknown>` — calls `xmlrpc/2/object` endpoint with `execute_kw(db, uid, apiKey, model, method, args)`
   - `createLead(data: LeadCreateData): Promise<number>` — high-level function that: gets config, authenticates, calls `execute_kw('crm.lead', 'create', [[data]])`, returns the new lead ID
   - `searchOrCreateUtmSource(name: string): Promise<number>` — searches for `utm.source` with given name, creates if not found, returns source_id. Uses `execute_kw('utm.source', 'search', [[['name', '=', name]]])` then conditionally `execute_kw('utm.source', 'create', [[{ name }]])`
   - Wrap XML-RPC calls in Promises (the xmlrpc library uses callbacks). Handle errors with descriptive messages prefixed with `[Odoo API]`.
   - Use `createSecureClient` when URL starts with `https://`, `createClient` otherwise.
   - Cache the uid after first auth to avoid re-authenticating on every request (module-level variable with TTL or simple cache).

5. Update `.env.example` — append Odoo section:
   ```
   # Odoo CRM Integration (server-only, do NOT prefix with NEXT_PUBLIC_)
   ODOO_URL=https://your-odoo-instance.com
   ODOO_DB=your-database-name
   ODOO_LOGIN=api-user@example.com
   ODOO_API_KEY=your-api-key
   ```

Follow project conventions: named exports, no semicolons, 2-space indent, camelCase file names, `@/lib/` path alias.

  <files>src/lib/validation/lead.ts</files>
  <action>
1. Install `zod`: `npm install zod`

2. Create `src/lib/validation/lead.ts`:
   - Import `z` from `zod`
   - Define `leadSchema` with these fields:
     - `name`: string, min 2 chars, max 100 chars, trimmed. Error: "El nombre debe tener entre 2 y 100 caracteres"
     - `email`: string, valid email format. Error: "Email no valido"
     - `phone`: string, regex `/^\+?[0-9\s\-()]{7,20}$/` to accept international formats. Error: "Telefono no valido (7-20 digitos, puede incluir +, espacios, guiones)"
     - `certification`: optional string, one of: "No certificado", "Open Water", "Advanced", "Rescue", "Divemaster", "Instructor". Error: "Certificacion no valida"
     - `preferredMonth`: optional string, one of the 12 month names in Spanish ("Enero" through "Diciembre"). Error: "Mes no valido"
     - `whatsappConsent`: boolean, must be true. Error: "Debe aceptar recibir comunicaciones por WhatsApp"
   - Export `leadSchema` and the inferred type `LeadFormData` using `z.infer<typeof leadSchema>`
   - Export a helper `validateLeadData(data: unknown)` that calls `leadSchema.safeParse(data)` and returns `{ success: true, data: LeadFormData } | { success: false, errors: Record<string, string> }`. On failure, flatten zod errors into `{ fieldName: "error message" }` format.

Error messages in Spanish to match the site language. All field names in camelCase following project conventions.
