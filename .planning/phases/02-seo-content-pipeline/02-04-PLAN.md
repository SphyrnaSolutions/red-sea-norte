  <action>
    1. Create `src/lib/seo/interlink-engine.ts`:
       - Import ClusterContext from cluster-resolver
       - Export `InterlinkResult` interface:
         ```typescript
         export interface InterlinkResult {
           relatedPages: Array<{ title: string; slug: string; url: string; role: string }>
           pillarLink: { title: string; url: string } | null
           contextualLinks: Array<{ title: string; url: string; anchor: string }>
         }
         ```
       - Export `computeInterlinks(clusterContext: ClusterContext | null, currentSlug: string): InterlinkResult`:
         - If clusterContext is null, return empty result (no cluster = no interlinks)
         - relatedPages: cluster siblings excluding current page, sorted by role (pillar first, then satellites)
         - pillarLink: link to the pillar page if current page is a satellite (null if current IS the pillar)
         - contextualLinks: up to 3 sibling pages formatted as anchor text suggestions (use page title as anchor text). These are meant for inline content links.
         - Limit relatedPages to max 6 items to avoid cluttering the page

    2. Create `src/lib/seo/keyword-map.ts`:
       - Export `KeywordMapEntry` interface: `{ keyword: string; slug: string; title: string; url: string }`
       - Export `async function getKeywordMap(): Promise<KeywordMapEntry[]>`:
         - Fetch all pages from Wagtail API (all types, fields: title, primary_keyword, cluster_id)
         - Filter to pages that have a non-empty primary_keyword
         - Return array of KeywordMapEntry objects
         - Cache with FetchConfig tags: ['keyword-map'], revalidate: 3600
       - Export `async function checkKeywordConflict(keyword: string, excludeSlug?: string): Promise<KeywordMapEntry | null>`:
         - Get the keyword map, find any entry where keyword matches (case-insensitive) and slug !== excludeSlug
         - Return the conflicting entry or null
         - This is a build-time/dev-time check -- primary enforcement is the database unique constraint (Plan 01)

    3. Create `src/components/seo/RelatedContent.tsx` as a Server Component:
       - Props: `{ interlinks: InterlinkResult }`
       - If interlinks has no related pages and no pillar link, render nothing (return null)
       - Render a section with heading "Contenido relacionado" (or "Te puede interesar")
       - Display pillarLink prominently if present (link back to pillar page with emphasis styling)
       - Display relatedPages as a grid of link cards (2-3 columns on desktop, 1 on mobile)
       - Each card: page title as link text, styled consistently with the site's existing card patterns (rounded corners, subtle shadow, hover state)
       - Use the site's color scheme: text-[#0D3A5D] for titles, border-[#D7E8EF], bg-[#FBFDFF]

    4. Update `src/lib/seo/index.ts` to re-export from interlink-engine and keyword-map
