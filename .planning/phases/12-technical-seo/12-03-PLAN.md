    Perform a targeted CWV optimization pass focusing on the three metrics:

    **LCP (Largest Contentful Paint < 2.5s):**
    - In layout.tsx: ensure no render-blocking scripts in <head>. The gtag consent default script uses beforeInteractive which is correct.
    - In next.config.ts: add Cache-Control headers for static assets (fonts, images) with long max-age:
      ```

      { source: '/_next/static/:path*', headers: [{ key: 'Cache-Control', value: 'public, max-age=31536000, immutable' }] }
      ```
    - Check if hero images use next/image with priority={true} for above-the-fold LCP element. If HeroSection uses a background image, ensure it has priority prop or fetchPriority="high".

    **CLS (Cumulative Layout Shift < 0.1):**
    - Verify font loading does not cause layout shift (should be fixed by Plan 01 with next/font/local)
    - Check if images in visible viewport have explicit width/height or fill prop with sizes attribute
    - In layout.tsx: ensure Header/Footer have fixed dimensions or min-height to prevent layout shift during hydration

    **INP (Interaction to Next Paint < 200ms):**
    - Review if any heavy JavaScript loads on interaction. The site is mostly server-rendered so INP should be low.
    - Ensure Framer Motion animations don't block main thread on interaction
    - Check that CookieConsent component is lazy-loaded or deferred (it's not critical path)

    **General optimizations in next.config.ts:**
    - Add image optimization config: set minimumCacheTTL to 2592000 (30 days) for optimized images
    - Remove i.pravatar.cc from remotePatterns if not actively used (reduces unnecessary image processing)

    Do NOT over-optimize or add unnecessary complexity. Focus on measurable improvements that affect the three CWV metrics.
